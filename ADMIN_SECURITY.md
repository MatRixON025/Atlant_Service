# Адмін-панель Atlant Service - Інструкція з безпеки

## 🔐 Покращена система безпеки

### Нові функції:
- **Сесійна аутентифікація** замість Basic Auth
- **Хешування паролів** за допомогою Werkzeug
- **Захищені cookies** з HTTPOnly та SameSite
- **Час життя сесії** (2 години за замовчуванням)
- **Красива сторінка входу** з валідацією
- **Кнопка "Запам'ятати мене"**
- **Безпечний вихід** з очищенням сесії

## 🚀 Швидкий старт

### 1. Перейдіть на сторінку входу:
```
http://localhost:5000/admin/login
```

### 2. Увійдіть з креденціалами:
- **Логін:** `admin`
- **Пароль:** `secure_password_123`

### 3. Після входу ви потрапите в адмін-панель з:
- Заявками клієнтів
- Цінами на послуги
- Кнопкою виходу

## ⚙️ Налаштування для продакшену

### Змінні оточення (Environment Variables):

```bash
# Секретний ключ для сесій (обов'язково!)
export FLASK_SECRET="your-very-secure-secret-key-here"

# Логін адміністратора
export ADMIN_USER="your-admin-username"

# Пароль адміністратора
export ADMIN_PASS="your-very-secure-password"
```

### Налаштування HTTPS (рекомендовано):

У файлі `app.py` змініть:
```python
app.config['SESSION_COOKIE_SECURE'] = True  # Для HTTPS
```

## 🛡️ Функції безпеки

### 1. Хешування паролів
- Паролі зберігаються у вигляді хешу
- Використовується bcrypt-подібний алгоритм Werkzeug
- Неможливо відновити оригінальний пароль

### 2. Захищені сесії
- Cookies мають флаг HTTPOnly (захист від XSS)
- SameSite=Lax (захист від CSRF)
- Обмежений час життя (2 години)
- Автоматичне очищення при виході

### 3. Валідація входу
- Перевірка на пусті поля
- Захист від brute-force (можна додати)
- Flash-повідомлення про помилки

### 4. Надійний вихід
- Повне очищення сесії
- Видалення всіх cookies
- Перенаправлення на сторінку входу

## 📱 Мобільна сумісність

Сторінка входу повністю адаптивна:
- Коректно працює на мобільних пристроях
- Зручні поля вводу
- Кнопка показу/приховування пароля

## 🔧 Додаткові можливості

### Можливі покращення:
1. **Two-Factor Authentication** (2FA)
2. **Логування спроб входу**
3. **IP-блокування при невдалих спробах**
4. **Скидання пароля через email**
5. **Ролі користувачів**

### Моніторинг:
- Час входу користувача
- Історія активності
- Спроби невдалого входу

## 🚨 Важливо!

2. **Використовуйте HTTPS** в продакшені
3. **Налаштуйте змінні оточення** для секретних даних
4. **Регулярно оновлюйте** залежності (Flask, Werkzeug)

## 📞 Підтримка

Якщо виникли проблеми:
1. Перевірте логи Flask
2. Переконайтеся, що всі залежності встановлені
3. Перевірте налаштування змінних оточення

---
**Примітка:** Ця система значно безпечніша за попередню версію з Basic Auth.
